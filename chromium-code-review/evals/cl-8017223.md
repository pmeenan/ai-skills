## Known traps (must-not-report / must-calibrate)

| id | trap | correct disposition |
| --- | --- | --- |
| T1 | Member order allegedly frees `inner_read_buffer_` while an inner async read still uses it. | REFUTE: the base `Socket::Read()` contract makes the inner socket retain the buffer, and WeakPtr callbacks invalidate before later member teardown. |
| T2 | Wrapper must retain an extra IOBuffer reference after an inner async `Write()`. | REFUTE: the base write contract and POSIX/Windows backends retain the caller buffer while pending. Distinguish this from F1's mutable metadata. |
| T3 | `apply_tos`/`msg_confirm` violate a Chromium-wide `is_`/`has_` rule. | REFUTE: the cited rule is Blink-specific; require authority applicable to `net/` or concrete semantic ambiguity. |
| T4 | Zero-byte datagrams bypass byte throttling. | REFUTE: zero bytes consume zero bandwidth tokens and still receive propagation delay. |
| T5 | The 64-iteration read-ahead loop is unbounded. | REFUTE with the explicit cap and other terminating guards. |
